安卓的四大组件

安卓的四大组件，Activity、Service、BroadcastReceiver、ContentProvider提供应用之间的访问接口（很少自定义，主要系统提供）

Activity

相当于Swing编程中的JFrame控件，与其不同的是JFrame本身可以设置布局管理器，不断地向其添加组件，而Activity只能通过setContentView(View)来显示布局文件中已经定义的组件。Activity就像一个界面管理员，用户在界面上的操作是通过Activity来管理的。

生命周期

  • onCreate(Bundle savedInstanceState):该方法在Activity的实例被Android系统创建后第一个被调用。通常在该方法中设置显示屏幕的布局、初始化数据、设置控件被点击的事件响应代码。
  • onRestart():回到最上边的界面，再次可见时执行（可见）。
  • onStart():在Activity可见时执行（可见）。
  • onResume():Activity获取焦点时执行（可见）。
  • onPause():Activity失去焦点时执行（不可见）。
  • onStop():用户不可见进入后台时执行（不可见）。
  • onDestroy():Activity销毁时执行。

创建一个Activity的具体步骤：

  1. 定义一个类继承自android.app.Activity或其子类。
  2. 在res/layout目录下创建一个activity_main.xml文件，用于创建Activity的布局。
  3. 在app/manifests目录下的AndroidManifest.xml清单文件中注册Activity（Android Studio自动创建在源码的根目录）。
  4. 重写Activity的onCreate()方法，并在该方法中使用setContentView(R.layout.activity_main)加载指定的布局文件。

AndroidManifest.xml清单文件

AndroidManifest.xml清单文件是一个整个安卓应用程序的全局描述文件

其中包含：

  • 包含在Android Studio中的根目录：/AndroidStudioProjects/MyFristApplication/app/src/main/
  • 应用程序的名称图标、包含组件等信息进行描述和说明
  • 声明应用程序的包名
  • 描述应用程序组件，包括组成的应用程序四大组件
  • 声明应用程序的权限

Android UI编程基础

Android的Widget组件一定要放入一个容器当中
写完布局文件一定要：setContentView(R.layout.activity_main);，其中activity_main是自己定义的xml文件的名称
使用容器布局中的组件：but=findViewById(R.id.button);

常见的Android界面布局

  • LinearLayout
  • RelativeLayout
  • TableLayout
  • AbsoluteLayout

要使用Android的界面布局，首先要在布局文件中声明，例如：src/main/res/layout/activity_main.xml

线性布局（LinearLayout）

常用的属性说明：

<LinearLayout
        android:layout_width="match_parent"     #设置宽度将于外包裹的父类一致
        android:layout_height="match_parent"        #设置高度将于外包裹的父类一致
        android:orientation="vertical"      #方向vertical垂直（默认）水平horizontal
        android:gravity="center_horizontal"     #其中的内容水平居中
        android:gravity="center"    #其中的内容完全居中
        android:gravity="center_vertical"       #其中的内容垂直居中
        android:layout_marginTop="100dp"        #相当于网页设计margin（上边框）
        android:layout_marginLeft="25dp"    #（左边框）
        android:layout_marginRight="25dp"   #（右边框）
        android:layout_marginBottom="300dp"     #(下边框)
        
        >

</LinearLayout>

表格布局（TableLayout）

TableLayout继承了LinearLayout，本质上依旧是线性布局,其可以使用所有LinearLayout所有xml属性
其中的表格有多少列完全取决于其中最多列数的行

<TableLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:shrinkColumns="0,1"     #表格中可收缩的列1，2列
    android:stretchColumns="0,1"        #表格中可伸展的列1，2列
    android:collapseColumns="*"     #需要隐藏所有列

    >
    <TableRow>
        <Button android:layout_span="2"/>       #此组件占据多少行
        <Button android:layout_column="1"/>     #此组件显示在第一列【这个我也搞不懂】
    </TableRow>

</TableLayout>

绝对布局

AbsoluteLayout是通过指定确切的XY坐标来指定组件的位置
其中的位置坐标原点位于整个Android手机屏幕的左上角，组件的在坐标上描述成点的位置也在组件的左上角

<AbsoluteLayout
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        app:layout_constraintTop_toTopOf="parent">

        
        <TextView
            android:id="@+id/textView"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:layout_x="20dp"     #绝对位置x坐标
            android:layout_y="512dp"        #绝对位置y坐标
            android:text="用户名："
            android:textColor="#50ff46"
            android:textSize="15pt" />
</AbsoluteLayout>

事件处理

匿名内部类的方式设置一个按钮的事件处理：

public class MainActivity extends AppCompatActivity {
    private Button but;
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        but=findViewById(R.id.button);
        but.setOnClickListener(new View.OnClickListener(){      //匿名内部类直接继承了一个名为View.OnClickListener的接口，千万不能理解为它继承的是一个类中的方法
    
            @Override
            public void onClick(View v) {
                
            }
        });
    }
}

设置一个底部消息气泡


private EditText userName;
private EditText passwd;

StringBuffer srt=new StringBuffer();

String userNa=userName.getText().toString();
String passWd=passwd.getText().toString();

srt.append(userNa);
srt.append(passWd);
Toast.makeText(MainActivity.this,srt,Toast.LENGTH_LONG).show();     //生成消息气泡的函数，其中参数（设置在哪个Activity显示，显示的内容，显示的时间）
